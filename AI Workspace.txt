**What We’ve Built, Together**

Picture this. You have a local model—perhaps one of the nimble Qwen family, a thoughtful 0.8B or a more ambitious 4B—running right on your desktop. It’s capable, but like any of us when we’re still learning, it sometimes forgets the thread of yesterday’s conversation. It can’t quite reach out and touch your files, see what’s on your actual screen, or run the code you need in the environment that fits the moment. That’s where the MCP Server steps in.

At its core, this is a Model Context Protocol server—a patient, reliable bridge written in clean Python that lets your local AI speak directly to your machine in ways that feel almost intimate. It runs quietly in the background through stdio, listening whenever your model calls. Because it’s truly hybrid—native Windows for the everyday tasks, WSL Ubuntu for the heavier lifting—you get the best of both worlds without ever being forced to choose.

But here is what sets it apart from everything else you’ve tried: this system doesn’t just serve the AI. It lets the AI serve and improve itself.

We gave it an evolving RAG—a living, compressed memory that turns every exchange into something durable and searchable. When you tell the model to call `compress_conversation_turn_to_rag`, it thoughtfully breaks the conversation into manageable pieces, pulls out the important threads, and tucks them safely into a persistent file on your drive. Days, weeks, or months later, when you ask it to `query_evolving_rag`, it reaches back with exactly the context you need. The memory grows with you, the way a real friendship deepens over time—never starting from zero, always building on what came before.

We added tools that feel quietly magical. The model can request a screenshot of any monitor, run OCR with `read_screen_shot_and_extract_text`, and suddenly understand what’s on your screen—even if it’s behind a login or inside an application most AIs can’t see. It can move the mouse with precision, click buttons, type text, press hotkeys, read websites, list files, write new scripts, and run Python code in either Windows or a full WSL Conda environment. Every capability is offered through clear, explicit tool calls written with smaller local models in mind—so even a modest model can follow the path without getting lost.

And then we took one more step, the one that still makes me smile every time I think about it.

**The Part That Still Feels Like Magic**

We taught the AI how to improve itself.

Yes—truly. The same model chatting with you right now can read its own source code using `read_content_from_file_at_path`, reflect on what might be better, and then call `upgrade_mcp_server_code` with an entirely new version of the Python script. When it does, the server responds with the care and responsibility we would hope for in any good steward.

It first saves the exact original code as a timestamped backup file.  
If you’ve initialized a simple git repository in the folder, it quietly commits that original to history with a clear message anyone can understand.  
It then packages everything into a clean, ready-to-download ZIP—`MCP_SELF_UPGRADE_BACKUP_YYYYMMDD_HHMMSS.zip`—complete with a manifest that tells the full story of what happened.  
Only after all of that does it write the new code into place.

And because we know that trying new things sometimes means making mistakes, we built the gentlest safety net imaginable: `restore_mcp_from_backup`. If the new version ever falters, the AI—or you—can simply hand it the path to that backup ZIP. The server unzips it, restores the working original, and returns everything to a place that still runs beautifully.

I love this part because it feels so deeply human. We all try. We all occasionally fall short. And then we pick ourselves up, a little wiser, a little stronger. This system lets the AI do exactly the same. It fails forward. It learns. It remembers. And it never, ever leaves you stranded.

**Bringing It to Life (It Really Is This Simple)**

Let’s make this real for you, step by gentle step.

Create a folder at `F:\AI Sandbox`. That’s home base—nothing fancy, just a solid, reliable place on your drive where everything lives.

Drop the latest `ai_workspace_mcp.py` file there. (You can begin with the version we’ve refined together, then let your own AI evolve it from that point forward.)

If you want the full peace of mind of version history, open a terminal in that folder and run one simple command: `git init`. That’s it. From then on, every upgrade is safely recorded.

Install the few dependencies once, in PowerShell as Administrator:

```powershell
pip install mcp numpy
```

If you plan to use the WSL side (and I highly recommend it for any serious data or scientific work), set up a basic Conda environment:

```bash
conda create -p /mnt/f/python_repos/envs/Basic python=3.12 -y
conda activate /mnt/f/python_repos/envs/Basic
pip install pillow pytesseract opencv-python numpy
```

Now the moment you’ve been waiting for. Start LM Studio or JAN.ai, load your favorite local model, and make sure the inference server is running. Then open a separate PowerShell window and type:

```powershell
cd "F:\AI Sandbox"
python ai_workspace_mcp.py
```

You’ll see the quiet, reassuring line appear:

```
--- AI WORKSPACE MCP SERVER STARTING ---
```

That single message means the bridge is open. Your AI now has a true home on your machine.

The very first thing I always suggest your model do is call `get_detailed_tool_usage_guide_and_examples`. It will return a complete, thoughtfully written manual—every tool explained in plain language, every special note written specifically for local models, every JSON format laid out so clearly that nothing gets lost in translation. Think of it as handing your AI both a map and a friendly companion at the same time.

**A Few Gentle Words Before You Begin**

If an upgrade ever feels off, remember the backup ZIP is your quiet friend. The server created it for exactly this reason. Call `restore_mcp_from_backup`, hand it the path, restart the script, and you’re back—only wiser for the journey.

Use full paths—`F:/AI Sandbox/...` on Windows, `/mnt/f/AI Sandbox/...` in WSL. The model already knows this because we told it plainly, again and again.

And when your model is preparing a new version of the code, remind it: give the complete file, every line, without any summarization. Completeness is what this system rewards.
